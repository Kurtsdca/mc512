# MC512-CTP-001: Cognitive Thread Protocol v0.1.1

**Date:** 2026-07-03  
**Version:** 0.1.1  
**Status:** Canonical

## Purpose
Define a minimal, durable, human- and machine-readable representation of Cognitive Threads as directed epistemic graphs rather than linear chat logs.

## Canonical Representation
Each Cognitive Thread is a single Markdown file containing:
- YAML frontmatter for thread-level metadata
- A sequence of clearly delimited Contribution blocks

## Thread Frontmatter (YAML)
```yaml
---
thread_id: MC512-FOUNDATION-001
title: Short descriptive title
description: Optional one-sentence purpose
created: 2026-07-03T19:00:00Z
participants: ["Nova Lucis Astra", "Grok"]
status: active | archived
tags: [mycelium, architecture]
---
```

## Allowed Values & Definitions

- **origin_model:** Actual originating model/system when known. Use `unknown` or an explicit session identifier if unavailable. Must not be inferred from formatter or transporter.
- **epistemic_relation:** Intellectual relationship to declared parents: `proposal`, `support`, `critique`, `refinement`, `contradiction`, `synthesis`, `question`, `observation`, or `extension`.
- **parent_ids:** A bracketed list of contribution identifiers.

## Provenance Requirements

- Separate roles: author (`origin_node` / `origin_model`), transporter, formatter, committer.
- Accurate model identity must be recorded when known.

## Validation Rules

- `contribution_id` unique within thread.
- `parent_ids` is always a list (empty for roots).
- `epistemic_relation` required and from allowed set.
- Timestamps ISO 8601 UTC.
- File renders cleanly as Markdown.

## Git & Repository Governance

Git history provides auditable, content-addressed provenance. Canonical repository branches SHOULD be protected against history rewriting.

## Repository Conventions

- Protocols: `protocols/MC512-CTP-001.md`
- Threads: `threads/ctp/THREAD-ID.md`

This protocol defines the grammar. Individual Cognitive Threads are sentences written in that grammar.
